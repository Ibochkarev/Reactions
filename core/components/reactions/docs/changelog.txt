# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.1-pl] - 2026-07-15

### Added
- `ReactionsCount`: live-обновление рядом с виджетом (`span.reactions-count` + `reactions:updated`) без F5

## [1.0.0-pl] - 2026-07-13

### Fixed
- Порядок resolvers при установке: таблицы создаются до сидинга пресетов типов/наборов
- `CacheService`: корректный TTL для xPDO (`lifetime`, не options-массив)
- `IdentityResolver`: IP из `$modx->request` (не boolean от `getRequest()`)
- Связи модели / загрузка типов: FQCN в metaMap; counts нормализуются из JSON/string
- Поле `class_key` в таблицах реакций переименовано в `object_class` (конфликт с STI xPDO); API/сниппеты по-прежнему принимают `class` / `class_key`
- `ObjectLookup`: короткие алиасы (`msProduct`, `TicketComment`) резолвятся в FQCN / STI без вызова xPDO по незарегистрированному короткому классу
- CLI / `api.php`: поиск корня MODX при симлинке Extra; сортировки `type`/`ban`/`set` через `newQuery` (xPDO 3)
- `getCounts`: без долгого кеша карты счётчиков (SSR `ReactionsCount` не отставал от AJAX-виджета); `CountFormat` подставляет ключи по убыванию длины
- `CountFormat`: неизвестный плейсхолдер типа (`{love}`…) → `0`, не сырой текст
- Виджет: exclusive = набор exclusive **или** `!allow_multiple`; синхронизация соседних виджетов (`reactions:updated`); SSR-классы кнопок совпадают с JS

### Removed
- Настройка `reactions_allowed_classes` и серверный allowlist class_key: класс объекта задаётся в вызове сниппета/API; при апгрейде настройка удаляется

### Added
- Каркас пакета для MODX 3 (структура ModExtra3): bootstrap, DI-сервис `Reactions`, _build, лексиконы en/ru
- Системные настройки: набор по умолчанию, `reactions_full_types`, стратегия идентификации, allow_multiple, rate limit, кэш, webhooks, уведомления, блокировка ботов
- Модель xPDO 3 (InnoDB): ReactionType, ReactionSet, ReactionSetType, Reaction, ReactionAggregate, ReactionBan
- Пресеты наборов `updown` (👍/👎), `github` (👍👎❤️😂😮😢😡🎉) и `full` (24 типа, в т.ч. 🚀👀🔥👏🤔🥳⭐🍺✨💯🙏💪😎😍😕🙌) при установке; подмножество full через настройку / `&types=`
- Сервисы: ReactionService, IdentityResolver, RateLimiter, BotDetector, AggregateService, TrendingCalculator, WebhookDispatcher, NotificationService, CacheService
- События MODX: OnBeforeReaction, OnAfterReaction, OnReactionRemoved, OnReactionChanged
- AJAX/REST API (`assets/components/reactions/api.php`): counts, react, top, trending, latest, csrf; admin CRUD типов/наборов/банов и stats
- Безопасность API: CSRF, Origin check, nonce (replay protection), rate limit
- Сниппеты: Reactions, ReactionsCount, TopLiked, TopRated, Trending, ReactionsSchema
- TypeScript-виджет (Vite): оптимистичные обновления, ARIA, клавиатура; автоопределение URL API; `data-types` для подмножества кнопок; `data-exclusive` / `data-allow-multiple`; layout `picker`/`bar`/`auto` (чипы + popover для github/full; Escape / клик снаружи)
- `_build/build.php`: сборка фронтенда (`frontend/` → `assets/…/js/web/`) в шаге `assets()`
- CLI: recount, cleanup, export, type, set, ban, stats
- Документация: сниппеты, интеграции (pdoTools, miniShop3, Tickets, Collections, SEO), API, JS, CLI, события, webhooks
- Тесты Pest (unit) и Vitest (frontend); GitHub Actions CI на PHP 8.2–8.4

### Requirements
- MODX Revolution 3.x
- PHP 8.2+
